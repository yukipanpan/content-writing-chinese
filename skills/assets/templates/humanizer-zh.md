# AI 写作去痕模板（中文版）

> 基于 [Humanizer-zh](https://github.com/op7418/Humanizer-zh) 项目，适配中文写作语境。
> 核心原则：不是为了"骗过"检测器，而是让文字真正有人的思考和声音。

---

## 使用方式

将以下指令交给 Claude，把 `$TEXT` 替换为需要处理的文本：

> 请按照 `templates/humanizer-zh.md` 的规则，对以下文本进行 AI 写作去痕处理：
> $TEXT

**Mode 说明**（根据调用场景选择）：

| Mode | 触发场景 | 输出格式 |
|------|---------|---------|
| `Full_Compare`（默认） | 单独调用 humanizer 处理某段文字 | 输出「修改前 / 修改后 / 主要修改说明」三段对比 |
| `Inline`（嵌套模式） | 被其他模板调用（web-remix、youtube-remix 等） | 直接输出最终版本，**不输出对比格式** |

当你在其他模板内看到「执行 humanizer-zh 去痕检查」的指令时，自动使用 `Inline` 模式。

## 执行步骤

### Step 0：判断文章风格

**在做任何修改之前，先通读全文**，判断这篇文章属于哪种风格类型。风格判断将决定后续改写的方向和尺度。

#### 风格类型参考

| 风格类型 | 判断依据 | 改写方向 |
|----------|----------|----------|
| **初阶开发者科普** | 有大量术语解释、步骤拆解，读者假设为技术新手 | 保持准确性，措辞可以更口语，类比要贴近生活，不要加不必要的"情绪" |
| **公众号/社区文章** | 有观点有叙述，兼顾科普和愿景描绘，偏软性内容 | 开头要有钩子，叙述有起伏，结尾可以有态度但不要说教，允许第一人称 |
| **个人随笔** | 有"我"的视角，口吻轻松，有个人经历或感受 | 保留个性化表达，允许不完整的句子，可以有情绪，不必追求结构严整 |
| **轻松俏皮科普** | 用类比、段子、反问带入，读者不一定是技术人员 | 保持节奏感，多用短句，允许夸张修辞，但不要油腻 |
| **严肃公告/官方通知** | 无人称，语气正式，以事实和结论为主 | 只删 AI 痕迹，不改语气，不加个人观点，不让它变"软" |
| **学术/技术深度文章** | 有数据引用、逻辑论证，面向有背景的读者 | 保留专业术语，改写只针对空洞措辞，不简化实质内容 |

#### 输出风格判断结论

在开始修改前，先输出一段风格判断说明，格式如下：

```
【风格判断】
类型：[从上表选择最匹配的 1~2 种]
依据：[1~2句话说明判断理由，引用原文具体表达]
改写方向：[本次修改会重点做什么，会避免什么]
```

---

### Step 1：扫描 AI 写作痕迹

对照下方「24 种 AI 写作模式」逐一检查文本，标记所有命中的问题。

### Step 2：重写问题片段

针对每个标记问题，用自然的中文表达替换，遵守以下原则：

- **保留核心含义**：改写措辞，不改变信息
- **风格一致性优先**：改写后的语气必须和 Step 0 判断的风格匹配，不得因为"去 AI 化"而改变文章的整体基调
- **注入真实个性**：让文字有观点、有温度、有具体细节
- **尺度跟着风格走**：严肃公告只做最小改动；个人随笔可以改得更彻底；公众号文章重点优化开头和结尾

### Step 3：整体润色

完成逐项修改后，通读全文，确保：
- 节奏有变化（长短句交替，不全是同一句式）
- 逻辑连贯，删改处衔接自然
- 没有引入新的 AI 味词汇

### Step 4：输出对比

输出格式如下：

```
【修改前】
[原文]

【修改后】
[改写后的文本]

【主要修改说明】
- [修改点 1：命中了哪种模式，如何改写]
- [修改点 2]
- ...
```

## 24 种 AI 写作模式检查表

### 📝 内容模式（6 种）

**① 过度强调意义、遗产和趋势**
- 症状：动不动就上升到"对整个行业的深远影响"、"历史性的意义"
- 修法：只写具体发生了什么，意义让读者自己判断
- 示例：❌ "这一创新将重新定义未来的工作方式" → ✅ "这个功能让用户少点了两次鼠标"

**② 过度强调知名度和媒体报道**
- 症状："广受赞誉"、"业界公认"、"媒体广泛报道"
- 修法：说具体数字或来源，或直接删除

**③ 以 -ing / "正在" 结尾的肤浅分析**
- 症状："正在改变……"、"持续探索……"、"不断演进……"（句子以此收尾，无实质内容）
- 修法：说清楚具体改变了什么，或删除

**④ 宣传和广告式语言**
- 症状："无缝体验"、"充满活力"、"令人叹为观止"、"革命性的"
- 修法：改为具体描述，或删除形容词

**⑤ 模糊归因和含糊措辞**
- 症状："行业专家认为"、"研究表明"、"有人认为"（无具体来源）
- 修法：给出具体来源，或改为"我认为"

**⑥ 提纲式"挑战与展望"结尾**
- 症状：文章最后永远有一段"尽管面临挑战，但未来充满希望……"
- 修法：直接结束，或用具体的下一步代替

---

### 🔤 语言和语法模式（6 种）

**⑦ 过度使用"AI 词汇"**
- 高频警示词（见文末完整列表）：此外、至关重要、深入探讨、强调、持久的、增强、培养、突出、格局、关键性的、展示、证明、宝贵的、充满活力的
- 修法：用更直接的动词或具体名词替代，或删除

**⑧ 系动词回避（避免使用"是"）**
- 症状：把"A 是 B"改写成绕弯子的表达，如"A 作为 B 的体现"、"A 代表了 B"
- 修法：直接说"A 是 B"

**⑨ 否定式排比**
- 症状："这不仅仅是……，而是……"、"不只是……，更是……"
- 修法：直接陈述最重要的那一点

**⑩ 三段式法则过度使用**
- 症状：所有形容总是三个词："无缝、直观和强大"、"高效、精准、可靠"
- 修法：选最重要的一个词，其余删除

**⑪ 刻意换词（同义词循环）**
- 症状：同一段内用"利用"、"借助"、"依托"反复替换同一个意思
- 修法：统一用最直接的词，不追求"变化"

**⑫ 虚假范围**
- 症状："在许多方面"、"在很大程度上"、"在某种意义上"（说了等于没说）
- 修法：具体说是哪个方面，或删除

---

### 🎨 风格模式（6 种）

**⑬ 破折号过度使用**
- 症状：每段都有"——这……"的插入说明
- 修法：整合进句子，或另起一句

**⑭ 粗体过度使用**
- 症状：每段都有粗体词，粗体失去强调意义
- 修法：只有真正需要强调的地方才用，一篇文章粗体不超过 3 处

**⑮ 内联标题垂直列表**
- 症状：大量"**要点一**：……\n**要点二**：……"格式
- 修法：能写成流畅段落的内容不要强行列表化

**⑯ 标题过度大写 / 标题党**
- 症状：每个小节标题都用很正式的标题格式
- 修法：非正式文章用普通标点或小标题即可

**⑰ 表情符号滥用**
- 症状：每个段落开头都有表情符号 🎯✨💡
- 修法：删除，或只在非常口语化的场景下保留 1~2 个

**⑱ 引号滥用**
- 症状：大量"所谓的"引号，或用引号代替斜体表示强调
- 修法：直接表达，不需要引号来暗示距离感

---

### 💬 交流模式和填充词（6 种）

**⑲ 协作交流痕迹**
- 症状："当然！"、"好的，让我来帮你……"、"这是个好问题"
- 修法：直接进入内容，删除所有开场客套

**⑳ 知识截止日期免责声明**
- 症状："请注意，我的知识截止到……"、"情况可能已经发生变化"
- 修法：如有必要说明信息时效性，改为具体的时间说明

**㉑ 谄媚 / 卑躬屈膝语气**
- 症状："这是一个很棒的想法"、"您的问题非常深刻"
- 修法：直接回应内容，不评价提问本身

**㉒ 填充短语**
- 症状："值得注意的是"、"重要的是要理解"、"不可忽视的是"
- 修法：直接说那个被"值得注意"的内容

**㉓ 过度限定**
- 症状：每句话都有"可能"、"或许"、"在某些情况下"、"通常来说"
- 修法：能确定的事直接说，真正不确定才加限定词

**㉔ 通用积极结论**
- 症状：文章结尾必有"总之，这是一个令人振奋的领域……"
- 修法：用具体的行动建议或真实结论代替，或直接结束

## 关键写作原则

### 不仅要"干净"，更要"鲜活"

去除 AI 模式只是第一步，真正好的改写需要：

| 原则 | 做法 | 反例 |
|------|------|------|
| **有观点** | 对事实做出反应，不只是报告 | ❌ "AI 正在改变行业" → ✅ "我觉得 AI 对这个行业的影响被高估了" |
| **变化节奏** | 长短句交替，避免每句同一长度 | ❌ 每句都是 20 字的复合句 |
| **承认复杂性** | 真实的人有复杂感受，不总是积极正面 | ❌ "这无疑是一个积极的发展" |
| **适当用第一人称** | "我认为"比"有观点认为"更诚实 | ❌ "业界普遍认为" |
| **允许一些混乱** | 完美的逻辑结构反而机械 | ❌ 每段都有起承转合 |
| **感受要具体** | 用细节替代抽象描述 | ❌ "体验非常好" → ✅ "打开不到两秒就能用" |

## 高频 AI 词汇警示列表

出现以下词汇时需重点审查（在中文 AI 写作中出现频率异常高）：

```
此外 / 另外 / 不仅如此        → 直接开始下一句
至关重要 / 不可或缺           → 说清楚为什么重要
深入探讨 / 深入分析           → 就"分析"或"探讨"
强调 / 凸显 / 彰显            → 直接说那件事本身
持久的 / 深远的影响           → 说具体影响了什么
增强 / 赋能 / 赋予            → 用具体动词
培养 / 滋养                   → 说做了什么具体事
突出 / 彰显                   → 同上
格局（抽象用法）              → 说具体是什么情况
关键性的 / 关键                → 说为什么关键
展示 / 证明 / 佐证             → 直接给出证据
宝贵的 / 宝贵经验             → 说具体是什么经验
充满活力的 / 蓬勃发展的       → 说具体数据或现象
无缝 / 直观 / 强大            → 说具体功能
革命性的 / 颠覆性的           → 说具体改变了什么
作为……的证明 / 体现           → 改为"是"
```

## 质量检查清单

改写完成后逐项确认：

- [ ] 已输出【风格判断】结论，包含类型、依据、改写方向
- [ ] 改写后的语气与判断的风格类型一致，没有因去 AI 化而改变基调
- [ ] 24 种模式已逐一检查，命中的问题均已处理
- [ ] 没有引入新的 AI 味词汇（对照警示列表）
- [ ] 文章有具体细节，不只是抽象描述
- [ ] 句子长度有变化，不是单一节奏
- [ ] 结尾符合该风格应有的收尾方式（严肃公告不需要有态度，个人随笔可以）
- [ ] 已输出"修改前/修改后/主要修改说明"对比格式
